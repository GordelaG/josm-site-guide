O que é Ground Layout?

O ground layout é a representação visual do pavimento aeroportuario utilizada pelos softwares de ATC no VATSIM. Criado a partir de dados do OpenStreetMap e processado no JOSM e QGIS.

JOSM

Editor avançado do OpenStreetMap para desenhar e classificar polígonos do aeroporto com precisão geográfica.

OpenStreetMap

Base de dados geoespacial colaborativa com tags específicas para infraestrutura aeroportuária mundial.

QGIS

Sistema de Informações Geográficas que processa e renderiza os polígonos importados para ferramentas ATC.

Classificação de Polígonos

Cada elemento do aeroporto deve ser mapeado com a tag OSM correta.
Filtre por categoria para encontrar o que precisa.

Sítio Aeroportuário

Infraestrutura
aeroway = aerodrome

Delimita o perímetro total do aeroporto. É o polígono base sobre o qual todos os demais elementos são desenhados.

ⓘ Desenhe primeiro — engloba toda a área do aeroporto

Área Militar

Infraestrutura
landuse = military

Delimita zonas de uso exclusivamente militar dentro ou adjacentes ao aeroporto. Comum em bases aéreas mistas.

ⓘ Utilize em SBNT, SBUL e bases da FAB

Gramado / Vegetação

Natural
landuse = grasslanduse = meadowlanduse = brownfield natural = woodnatural = scrubnatural = wetland

Áreas verdes, gramados, campos não pavimentados, vegetacao rasteira e zonas úmidas ao redor das pistas.

ⓘ Use multiplas tags conforme o tipo de vegetacao real

Água

Natural
natural = water

Corpos d'água como lagos, lagoas, reservatórios e rios que fazem parte ou margeiam a area aeroportuária.

ⓘ Importante para aeroportos como SBCT e SBFL

Área em Construção

Infraestrutura
landuse = construction

Zonas com obras em andamento, pavimentação nova ou reformas dentro do perímetro aeroportuario.

ⓘ Mantenha atualizado conforme progresso das obras

Pátios (Apron)

Operacional
aeroway = apron

Area pavimentada onde aeronaves ficam estacionadas, passageiros embarcam/desembarcam e carga é operada.

ⓘ Separe pátios de domestico, internacional e carga

Prédios / Edificações

Infraestrutura
building = yesaeroway = hangarbuilding = office building = publicbuilding = roofbuilding = transportation

Todas as construções: terminais, hangares, torres, edifícios administrativos e coberturas.

ⓘ Diferencie terminais de passageiros dos hangares

Pontes

Infraestrutura
man_made = bridge

Estruturas de conexão sobre obstáculos como vias, rios ou desníveis dentro da area aeroportuária.

ⓘ Aplique também em pontes de embarque (fingers)

Helipontos

Operacional
aeroway = helipad

Áreas designadas para pouso e decolagem de helicópteros, podendo ser no solo ou em coberturas de edifícios.

ⓘ Use como polígono ou nó, conforme o tamanho

Taxiway

Operacional
aeroway = taxiway

Vias de taxiamento que conectam pistas de pouso/decolagem aos pátios de estacionamento de aeronaves.

ⓘ Nomeie com ref= para identificar cada taxiway (ex: ref=A)

Gates / Posicoes

Operacional
aeroway = parking_position

Posicoes de estacionamento individuais de aeronaves, incluindo gates de terminal e posicoes remotas.

ⓘ Adicione ref= com o número do gate (ex: ref=1, ref=A1)

Tutorial Passo a Passo

Siga este guia para criar seu primeiro ground layout do zero.

Configurando o JOSM

⬇️

1. Instalar o JOSM

Baixe a versão mais recente em josm.openstreetmap.de. Requer Java 11+.

🔌

2. Instalar Plugins

Vá em Editar → Preferências → Plugins e instale os seguintes plugins obrigatórios:

buildings_toolsvalidatorimagery_offset_dbopendata

(Opcional) Para habilitar o modo noturno/Dark Mode, instale também:

flatlaf
📁

3. Baixar Gabarito da Pista

Baixe o arquivo de gabarito abaixo e abra-o no JOSM para ter a referência exata do alinhamento da pista de pouso sobre o satélite.

Baixar runway_v2.kml

Baixando Dados do Aeroporto

🔍

1. Localizar o aeroporto

Use Arquivo → Baixar Dados (Ctrl+Shift+Down) e pesquise pelo código ICAO ou nome do aeroporto.

📦

2. Definir área de download

Delimite a área cobrindo todo o aeroporto com zona de segurança de ~500m ao redor do perímetro.

💾

3. Salvar localmente

Salve o arquivo .osm localmente antes de editar. Mantenha backups regulares durante o trabalho.

Mapeando os Elementos

✈️

1. Comecar pelo aerodromo

Desenhe o polígono externo com aeroway=aerodrome. Este será a base de todos os outros elementos.

🛫️

2. Mapear pistas (runways)

Desenhe cada pista com aeroway=runway, adicionando ref= com a designação (ex: 10/28).

🛣️

3. Adicionar taxiways

Mapeie as vias de taxiamento com aeroway=taxiway e ref= com a letra identificadora.

🏘️

4. Mapear pátios e edificacoes

Adicione pátios (aeroway=apron), terminais (building=transportation) e demais estruturas.

Validando as Tags

✅

1. Executar validador

Use Ferramentas → Validar (Shift+V) para identificar erros de topologia e tags incorretas.

🔗

2. Verificar conectividade

Certifique-se que todas as taxiways conectam corretamente as runways e pátios do aeroporto.

📋

3. Checklist de tags obrigatórias

Todo elemento deve ter: tag principal, name= quando aplicável, ref= para pistas/taxiways/gates.
